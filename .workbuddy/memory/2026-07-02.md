## 2026-07-03 注册接口排查
- 现象：POST /signup 返回 `{"msg":"success"}` 但 user 表无新记录。
- 根因：`dao/mysql/user.go` CheckUserExist 第19行 SQL 写成 `from users`（复数），而建表脚本 `models/create_table.sql` 中表名是 `user`（单数），InsertUser 用的也是 `user`。查询不存在的表直接报错，SignUp 在第一步就 return，从未执行 insert。
- 加剧因素：原 controller/user.go 第35行 `logic.SignUp(p)` 丢弃返回的 error 并无条件返回 success，导致静默失败、难以定位。（用户后续已自行补上 error 判断）
- 修复：CheckUserExist 的 `users` → `user`。go build 通过。
- 附注：encryptPassword 中 h.Sum 被调用3次、前两次结果丢弃，逻辑上等价于 hex(password字节 + md5(password))，属设计问题但非本次阻塞。

## 2026-07-03 登录接口排查
- 现象：POST /login 始终返回"密码错误"，即使密码正确。
- 根因：`dao/mysql/user.go` Login 第60行判断条件写反，`if password == user.Password { return errors.New("密码错误") }`，密码相等时报错。
- 修复：`==` → `!=`。go build 通过。
- 期间用户已自行把 encryptPassword 改为 `hex.EncodeToString(h.Sum(nil))`（标准 md5 hex），调用更干净。
- 注意：加密算法变了，旧库中按旧算法存的密码会失配。若 user 表里已有旧数据需重新注册或回填。
